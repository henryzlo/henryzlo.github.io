Vision, Language, and the Big Problems in AI

We discuss some advances in computer vision, language processing, and the big problems facing AI.

Computer Vision

  • Computer vision deals with processing images, and sometimes movies, to perform intelligent tasks.
  • Often, images need to be preprocessed before handled.
    • For example, recognizing building outlines requires that we first perform edge detection.
      • Example of edge detection:
      • Idea: find those points on the image where the color changes sharply.
      • Also useful for identifying things based only on their outlines or shapes.
    • In optical character recognition, we often have to center the image, scale, rotate, and despeckle it first.
  • Now images are preprocessed, we can do "intelligent" things related to images.
  • Object detection / recognition is a classification problem.
    • Features need to be extracted first (e.g. color, shape, size, angle).
    • These features are fed into a trained classifier, which then tells us what kind of object it is.
    • For example, for face detection, we may want features which are characteristic of faces, and when we give it to a classifier of faces, it will tell us whether we have a face or not.
  • Object tracking is the problem of tracking an object in an image over time.
    • This first requires object recognition - we have to recognize an object before we track it.
    • Then we have to solve the problem of determining whether the object in one frame of video is the same as the object in another frame of video.

Natural Language Processing

  • Actual language understanding is difficult to achieve, because language is complex.
    • Grammar can be complicated.
    • Rules are not always followed.
    • Words, sentences, and phrases can be ambiguous or have multiple meanings.
  • Because of this, tasks which require actual understanding are much more difficult.
  • An example task: find documents which are related to each other.
    • This does not require actual understanding, and can be done quite well with only statistical techniques.
    • Suppose we have a set of documents. From each document, we might want to extract keywords.
    • Obviously we want to avoid common words such as "the", "is", etc.
    • We can then "measure" how closely related documents are based on how many keywords they share.
    • We may even be able to group together keywords into topics.
  • Some other, more difficult problems in natural language processing:
    • Word sense disambiguation:
      • This is the problem of determining which "sense" of a word we mean to use.
      • For example, consider the two sentences:
        The bass is too loud.
        We caught a bass today.
      • We know that the first sentence probably referred to bass as in bass and treble, because bass (the fish) are usually pretty quiet.
      • We know that the second sentence refers to the fish because you can't catch bass (the audio concept).
      • Some techniques:
        • Using pre-programmed "common sense", like we did in the example above.
        • Using a trained classifier (doesn't work very well).
        • Using the presence of commonly associated words nearby (e.g. caught).
    • Machine translation:
      • True machine translation of large amounts of text requires actual understanding of language. Current methods are only approximations of this.
      • Some techniques:
        • Rule-based systems, which contain mappings between the two language words, and their grammatical structures.
        • Translating each word one-by-one, then stringing them together into a somewhat cohesive sentence.
  • For most NLP problems, there are "deep" solutions and "shallow" solutions.
    • "Deep" solutions seek actual understanding of language, while "shallow" solutions use less than a full understanding (e.g. statistical techniques).
    • Currently, shallow techniques predominate - they are easier to code, easier to train, faster, and therefore more applicable to real life situations.
    • Arguably, shallow techniques are not actually "intelligent", whereas deep techniques are.

Strong AI vs. Weak AI

  • Weak AI refers to systems which are smart enough to perform a specific intelligent task, but may not truly understand.
    • Most techniques discussed - machine learning, problem solving, decision making, etc. - are weak AI techniques.
    • The current state of AI focuses on weak techniques.
    • For most cases - image recognition, etc. - weak techniques are good enough to be commercially viable.
  • Strong AI refers to creating a truly intelligent system, like a human mind.
    • Some problems, like large-scale machine translation, may only be solvable by strong AI systems.
    • Creating strong AI systems, however, take much more work than weak AI, and are slower.
  • Many researchers (I think) go into AI hoping to build strong systems, but end up working on weak ones.
    • It's easier to get funding for weak AI projects.
    • Projects often don't pan out - strong AI projects often take much more time than weak ones.
    • The subgoals of weak AI are much easier to define than the lofty goals of strong AI.
    • For many proposed strong AI systems, there is enormous effort involved (e.g. brain studies, simulating neural structures, etc.)
