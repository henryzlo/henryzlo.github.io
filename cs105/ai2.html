Artificial Intelligence

Artificial intelligence is an ancient goal (yet it always seems novel) to construct human-level intelligence.

Can Computers Think?

  • If we can specify any procedure in enough detail, then we can program it.
    • This is the Church-Turing thesis.
    • If we consider intelligence to be a procedure, then the thesis says that yes, computers can think.
    • Are there limitations to using code? Yes, but they are very theoretical. Mostly everything can be programmed.
  • The problem is how to define intelligence.
    • We discuss various ways of posing the problem of intelligence, and respective solutions.
    • These different formulations have led to very different methods of simulating intelligence.
    • Therefore AI has become a fairly fragmented field (maybe intelligence itself is a lot of things, rather than one concept).

Turing Test

  • The idea: It's intelligent enough if it can fool someone into thinking it is.
  • Alan Turing, famous computer scientist, proposed the Turing test to determine whether a program is intelligent or not.
    • The test is this: put a person in a room with a door. This person interacts with something behind the door using a keyboard.
    • If a computer behind the door can fool the person into thinking it is in fact human, then the computer has passed the Turing test.
  • This has led to the development of chatting programs, which were designed to fool humans.
    • Examples: Chatterbot, Cleverbot, Siri (kind of).
    • Modern programs are very good, able to fool people for at least for a few minutes.
    • Most do not actually learn - use tricks such as asking questions, using common answers.
    • Some applications: customer service.
    • Otherwise not that useful.

Machine Learning

  • The idea: intelligence learns.
  • The field of machine learning studies different methods to solve common problems.
    • Common problem: given some data, and some labels for this data, apply the correct labels to future data.
    • For example, show a program a bunch of pictures, and tell it which ones are faces and not.
    • It should be able to tell if future images are faces by itself.
    • This is called classification; other problems will be discussed next time.
  • Many different models exist, with different concepts of "intelligence".
    • Statistical methods are very popular: easy for mathematicians to understand and use, but whether it is truly "intelligent" is debatable.
    • Margin-based methods also work very well; they try to separate data of different labels, and put data in similar labels together.
    • Neural network methods try to emulate how brains work; brains store knowledge in connections, and neural nets do the same.
    • Decision trees conceptually are similar to how people make decisions (sometimes).
  • Machine learning is the most useful and active field in AI research.
    • Studied methods are very general, and can be applied to many problems.
    • Recent successes: computer vision, speech recognition, stock market prediction, e-Harmony, etc.

Decision Making

  • The idea: intelligence is making rational decisions given some input.
  • In the right environment, computers can be much better than humans in making decisions.
  • For example, game playing (some games).
    • Games can be represented in game states.
      • In the beginning, the state of the game is that there are no marks on the board.
      • The end states (there are multiple) consist of three marks of the same kind in a row, or all nine marks on the board.
      • At the beginning, there is one possible state.
      • We then progress to one of many states, and continue until we get to an end state.
    • Game states can be connected in a tree.
      • The root is the beginning of the game.
      • The leaves of the tree are the end of the game.
      • A single game is represented as a path from the root to a leaf.
      • The game of tic-tac-toe can be represented in a tree.

    • Given this game tree, a simple algorithm can be used to make the best decisions.
  • Programs can also prove mathematical theorems, something very difficult for humans to do.
    • In math (e.g. geometry), we have a very limited set of basic rules called axioms.
    • All theorems are derived from the axioms.
    • In this limited environment, automated theorem provers can easily come up and prove new mathematical theorems.
  • Programs can also deal with some uncertain situations well, e.g. playing cards.
    • We can know the probabilities associated with different cards and outcomes.
    • Programs can use this to make optimal decisions.
  • In general, programs can make optimal decisions, but only under very restricted conditions.
    • Knowledge of outcomes (e.g. chess, checkers) helps.
    • Programs cannot deal with unmeasurable uncertainty; for example, the millions of different things that can happen in every day life.
    • In our environment, outcomes are uncertain, rules are not well-specified, and there are many unknowable variables; current algorithms can't make good decisions in these environments.

Knowledge Representation

  • Idea: intelligence is being able to remember, recall, and connect pieces of information.
  • Example: Game trees store knowledge about a game, and can be used to make intelligent decisions.
  • Example: Graphs can be used to represent states in general, and this can be used to solve certain problems.
    • Nodes can represent concepts, relationships between concepts can be represented as edges (concept map).
    • Another example: songs are nodes, songs played next by people are edges.
    • Using this, a program can solve problems by searching - e.g. What songs are related to one another? What playlist might you like?.
  • Another graphical example: Bayesian networks, which model causal relationships.
    • Nodes represent causes, and an edge between nodes indicate potential causal effect.
    • In this example, the season may influence rain, and whether or not we use sprinklers.
    • Both rain and sprinklers can cause the ground to get wet.
    • Wetness can subsequently cause the ground to be slippery.
  • Example: decision trees.
  • Decision trees can be built for expert systems, which are computerized decision making machines.
    • For example, programs which automatically diagnose based on user input can use decision trees.
    • Another example, automated telephone systems are a form of expert system.
    • These types of systems are not very flexible because they can only handle what they have been scripted for.
    • However, by essentially not thinking, these types of systems are very fast.
  • Knowledge representation based approaches usually require a knowledge base.
    • Sometimes, this is just enough knowledge for the task at hand; some try to instill a "commmon sense" knowledge base.
    • This can be learned, or taught by people.
    • Problem with learning: computers do not have the same drives / motivations / experience as humans; may learn very different concepts.
