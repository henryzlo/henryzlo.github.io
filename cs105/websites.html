Websites

We discuss how websites work.

Web Page

  • Web pages contain multiple pieces of information, which are put together by a web browser.
    • Pieces include HTML, CSS, images, javascript, etc.
    • The act of putting together a web page for presentation is called rendering.
    • As mentioned, this may involve multiple HTTP requests.
    • The main pieces of a web page, from http://www.tedroche.com/Present/2010/css/HTML-CSS-JS-Layers-half.png:
  • HTML (hypertext markup language) controls the elements and content of a web page.
    • All text, links, including links to the images displayed on a web page, are in the HTML.
    • HTML is a markup language, not a programming language.
    • It's for describing what is shown, not writing programs.
    • HTML consists of tags, which can be nested within each other.
    • Each tag describes an element of the web page.
    • Example HTML, from http://upload.wikimedia.org/wikipedia/commons/8/84/HTML.svg:
    • This web page is an example of raw HTML - its not pretty.
  • CSS (cascading stylesheets) controls how things look on a web page.
    • For example, background colors, font sizes, colors, borders, etc.
    • Almost all purely stylistic and visual elements are due to CSS.
    • CSS is a set of descriptors, not a programming language.
    • Example CSS, from http://en.support.files.wordpress.com/2011/09/css-selectors-lrg.png
    • Style specifications used to be part of HTML, but this makes reading HTML very difficult so it is separated into style sheets.
  • Javascript controls advanced behaviors of a web page.
    • Anything from animations, instant chat, google maps, is all due to Javascript.
    • Javascript is a programming language.
    • Much of javascript is used to manipulate the web page.
    • The web page is represented as a tree called the DOM (document object model).

Web Applications

  • Static web sites serve up web pages directly from the file system.
    • This is an example of a static site.
    • The HTML on this page is saved as a file on the server, and the server presents this to the client.
  • Dynamic web sites create parts of the web site on demand from the client request.
    • Almost all complex web sites are dynamic.
    • For example, Facebook information differs depending on who's logged in.
    • Thus, Facebook creates web pages dynamically, based on who's logged in, among other things.
  • Dynamic web sites create parts of the web page using essentially a program.
    • Programming languages such as PHP or Python are used to code advanced behaviors of the web server.
    • These programs are run every time the request for the page is given.
    • The output of the program is an HTML web page.
    • Thus, HTML is generated on the fly, rather than merely retrieved.
    • Examples: your friends list cannot be saved as HTML because it changes all the time.
    • This information has to be updated every time.
    • The code on this page retrieves your user login information (session info) and generates your friends list based on this.
  • Dynamic web sites often store and retrieve information.
    • This information could be, for example, items on an online store.
    • Each item could have a description, pictures, reviews, etc.
    • This is complicated, yet structured, information.
    • This is stored in one or multiple databases.
    • The web application programs retrieve and store information on these databases as necessary.
    • Example: every time you go to an online store, the program to generate the page runs.
    • This program will need to look in the database for the current items to display.
  • Web application architecture diagram, from http://www.agileload.com/images/image/web-client-server-db283134E9F23C.gif?sfvrsn=2
  • Web servers handle the requests from clients and sends responses.
    • Web servers run continuously, listening for requests.
    • On receiving a connection, web servers parse this information to learn what clients want.
    • Web servers run the web application (program).
    • Web servers retrieve static files (for static web sites, or saved images, etc.) as necessary.
    • Web servers also manage system load, security, and other things for web sites.
