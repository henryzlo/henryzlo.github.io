Artificial Intelligence

Artificial intelligence is an ancient goal (yet it always seems novel) to construct human-level intelligence.

Artificial Intelligence

  • Since ancient times, people have wondered about building humanoid automatons.
    • Hephaestus, Greek god of craftsmen, created automatons out of metal to help him with his work in myth.
    • Yan Shi, artificer in the 3rd century of China, built a life-sized human machine to sing and dance.
    • al-Jazari, inventor in the 12th century Middle East, built machines to serve drinks, play music, and wash hands.
  • Programmable computers simplified developing AI.
    • Instead of having to build machines and robots, now artificial intelligence seekers only need to write code.
    • Essentially, with computers, engineers now face only the core problem: how to think?
  • Four different ways of approaching AI:
    Acting Humanly Acting Rationally
    Thinking Humanly Thinking Rationally
  • The "acting like a human" approach seeks to create computers which emulate humans.
    • For example, this approach may attempt to pass the Turing test.
    • The test is this: put a person in a room with a door. This person interacts with another entity through notes slipped under the door.
    • If a computer behind the door can fool the person into thinking it is in fact human, then the computer has passed the Turing test.
  • The "acting rationally" approach seeks to create computers which perform correctly in various situations.
    • Defining correct behavior is a lot easier than defining human behavior.
    • Thus, most work has been done on this approach. This is what we will focus on.
  • The "thinking humanly" approach seeks to understand human thought, and then use this knowledge to create intelligent machines.
  • The "thinking rationally" approach seeks to model the foundations of logical thought.
  • Traditionally, the goal of AI is to develop human-like intelligence.
    • Problems include: planning and reasoning.
    • Representing knowledge.
    • Learning.
    • Speech recognition.
    • Image understanding.
  • We will show some models of thinking, and then show problems with them, to illustrate the difficulty of AI.
    • We do not know what intelligence is (only know it by analogy).
    • In order to build an intelligent machine, we model some aspect of intelligence.

Reasoning as a Graph Search

  • Playing games intelligently is one of the classic problems of AI.
  • Games can be represented in game states.
    • In the beginning, the state of the game is that there are no marks on the board.
    • The end states (there are multiple) consist of three marks of the same kind in a row, or all nine marks on the board.
    • At the beginning, there is one possible state.
    • We then progress to one of many states, and continue until we get to an end state.
  • Game states can be connected in a tree.
    • The root is the beginning of the game.
    • The leaves of the tree are the end of the game.
    • A single game is represented as a path from the root to a leaf.
    • The game of tic-tac-toe can be represented in a tree.
  • Example (ignore the numbers):
  • The intelligent behavior in this situation is the one that gets us to win.
    • At each step in the game, we proceed one step down the tree.
    • The goal is to manipulate the path so that we get to a leaf which represents a "win".
    • Note that if we are player "X", we only get control of the "X" moves.
  • This type of representation can be used for other games as well (e.g. chess, checkers, etc.).
    • However, for complicated games like chess, the tree can get too large to fit in memory.
    • Efficient algorithms must do something to reduce the number of states processed, often sacrificing performance.
  • Other than games, graphs can be used to represent states in general, and this can be used to solve certain problems.
    • The pathfinding example: states represent places, connections represent paths, and the goal is to find the shortest path from one place to another.
    • Becoming CEO: states represent positions, connections represent ways of getting there, edge weights may represent difficulty, and the goal is to get to the CEO position from where you begin.
    • In general, state graphs can be used to represent decision-making problems if we know the outcomes beforehand.
  • Some problems, however, cannot be represented by graphs.
    • Problems without discrete states cannot be represented by graphs.
    • For example, in air hockey, the velocity of the puck, angle, position, etc. are all variables, too many states to represent in a graph.
    • Problems where you do not know the outcome cannot be represented by graphs.
    • Each state represents an outcome of some decision, so if you don't know the outcome of your decisions, you can't represent it.

Decision Trees

  • You've probably seen decision trees before:
  • Decision trees represent knowledge similarly to graphs, but how we use them is different.
    • Each node represents a decision we can make, rather than an outcome.
    • Depending on the decision we make, we follow the tree down to the next decision.
    • We keep doing this until we reach a conclusion.
    • Thus, the tree represents a decision making process, rather than a state space.
  • Decision trees can be built for expert systems, which are computerized decision making machines.
    • For example, programs which automatically diagnose based on user input can use decision trees.
    • Another example, automated telephone systems are a form of expert system.
    • These types of systems are not very flexible because they can only handle what they have been scripted for.
    • However, by essentially not thinking, these types of systems are very fast.
