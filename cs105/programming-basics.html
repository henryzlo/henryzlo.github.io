Programming Basics

Programming is the art of putting instructions together to do something useful. This lecture discusses the basic building blocks of code, which will later be used to build up algorithms and programs.

Data Types

  • Three categories of primitive data types: numerical, string, and boolean.
  • A numerical variable represents a number.
    • Some languages make a distinction between integers and floats.
    • Integers are whole numbers, like -1, 0, and 23.
    • Floats represent decimals and fractions like 0.6, 2.0/3.0, and pi.
    • In these languages, an expression involving two integers, like 2/3, will give you back an integer.
    • To get back a float, at least one of the numbers must be a float, like 2/3.0.
  • A string variable represents text, for example "2" and "asdf".
    • "2" and 2 are NOT the same: "2" is a string, and 2 is a number.
    • Some languages distinguish between a character and a string, which has multiple characters.
    • In these languages, a character is delineated with single quotes, e.g. '[' or 'a'.
    • A string is delineated with double quotes, e.g. "some string".
    • In this class, we do not make the distinction, and use single and double quotes interchangeably.
  • A boolean represents either true or false.
  • There are other data types, such as objects and arrays, but these are not primitive and will be discussed later.

Expressions

  • An expression is a piece of code that evaluates to a value.
    • For example, 3+2 is an expression that evaluates to the value 5.
    • Note the distinction between the expression (3+2) and its value (5).
    • Some expressions don't always return a value.
    • Three general kinds of expressions are numerical expressions, boolean expressions, and string expressions.
  • Numerical expressions always return a numerical value.
    • Example: (4+2)*5
    • Numerical expressions can be composed of other numerical expressions.
    • Numerical operators (such as +,-,*,/) create numerical expressions, e.g. 5/2 is a numerical expression.
    • Calls to numerical functions are numerical expressions, e.g. average(3,4).
    • In general, anything that results in a number is a numerical expression.
  • String expressions return a string value.
    • Anything resulting in a string is a string expression
    • String concatenation is a string expression - e.g. "hi" + " you" results in the string "hi you".
    • Calls to string functions are string expressions, e.g. substring("something",0,3) yields the string "some".
  • Boolean expressions return either true or false.
    • true and false, and anything that results in them, is a boolean expression.
    • Relational operators (==,<,>, etc.) result in boolean values, and thus are boolean expressions.
    • For example, 3 < 4, 5 == 5, 5 > 2 all are either true or false, and hence are boolean expressions.
    • Equality (==) is a relational operator, and can be used to create boolean expressions, e.g. "apples"=="oranges" and 3==3.
    • Boolean expressions can be created from other boolean expressions using boolean operators and, or, and not.
    • The output of these operators are given by truth tables.
    • Truth table for and:
      xyx and y
      truetruetrue
      truefalsefalse
      falsetruefalse
      falsefalsefalse
    • Truth table for or:
      xyx or y
      truetruetrue
      truefalsetrue
      falsetruetrue
      falsefalsefalse
    • Truth table for not:
      xnot x
      truefalse
      falsetrue
    • Example: (5+3) > 6 is a boolean expression which evaluates to true
    • Example: 3==4 or 5==3 is a boolean expression which evaluates to false
    • Example: (not 7==4) and (not 6==3) is a boolean expression which evaluates to true
  • Function calls are always expressions.
    • Function calls look like function_name(input), where function_name is the function name, and input is the input to the function.
    • For example, print("Hi") is a call to the print function, and the string "Hi" is the input fed to the function.
    • Functions can have multiple or no inputs, for example average(4,5) and ls().
    • Functions return either one value (numerical, string, boolean, or other) or nothing.
    • For example, delete(filename) returns nothing, and average(4,5) returns a number.
    • Function calls always have a function name, followed immediately by an open parens (, then its inputs, if any, and finally ended by a close parens ).

Statements

  • A statement does NOT return a value, but does something.
    • Some expressions, such as function calls, are also statements.
    • Every statement is one or more lines of code.
    • Every expression is always inside a statement.
  • Assignment statements create a variable to store some value.
    • Assignment statements look like variable_name = some_expression. some_expression is ALWAYS some kind of expression.
    • After this statement is run, variable_name now stores the value of some_expression.
    • For example, a_variable=4+6 assigns the value 10 to a_variable.
    • a_variable + 5 now yields 15, and print(a_variable) will print out 10.
    • A common type of assignment statement looks like var = var + 1.
    • This takes the current value of var, adds one to it, then assigns that value back to the variable var.
    • For example, if var was 5 before this line of code runs, then var will be 6 after it.
  • if statements, while loops, and for loops are called flow control statements
    • So called because they make the program skip or repeat some lines of code.
  • if statements branch, depending on a boolean expression.
    • Always look like:
      if (boolean_expression)
      {
        // some code
      }
      else
      {
        // other code
      }
    • boolean_expression MUST be a boolean expression.
    • If boolean_expression evaluates to true, then the first block of code (some code) is run.
    • If boolean_expression evaluates to false, then the second block of code (other code) is run.
    • Exactly one of the two blocks of code is run.
    • The else part and second line of code are optional. In this case, if boolean_expression is false, then we don't execute the first block of code, and then continue as normal.
    • Example:
      if (true)
      {
        print("its true!")
      }
      else
      {
        print("its false!")
      }
      always prints "its true!"
    • Example:
      if (x>70)
      {
        print("you pass")
      }
      else
      {
        print("you fail")
      }
      prints "you pass" if x is 70 or less, and prints "you fail" otherwise.
    • There are other data types, such as objects and arrays, but these are not primitive and will be discussed later.
  • while loops keep running some code while a boolean expression is false.
    • Always look like:
      while (boolean_expression)
      {
        // some code
      }
    • boolean_expression is always a boolean expression.
    • If boolean_expression returns true, then the block of code (some code) is run, otherwise, we skip the block.
    • After the block of code is run, we check the boolean_expression again. If it is true, then run block of code again, etc.
    • Example:
      i=3
      while (i>0)
      {
        i=i-1
        print(i)
      }
    • When we first enter the loop, i is 3. So we reduce i by 1, then print it. The first run prints a 2.
    • Now we check the boolean expression again. Now i is 2, which is still greater than 0. So we run the block of code again, which reduces i to 1, then prints it.
    • i is now 1, which is still greater than 0. Reduce to 0 and print.
    • Finally the boolean expression is false: i is not less than 0.
    • We skip the block of code (break out of the loop) and do whatever comes next.
  • for loops usually are a more concise form of a while loop.
    • Looks like:
      for (assignment_statement; boolean_expression; statement)
      {
        // some code
      }
    • assignment_statement is always an assignment statement; boolean_expression is always a boolean expression; statement is usually an assignment statement.
    • assignment_statement is run only once, before the loop is entered.
    • statement is run every time AFTER the code in the loop is run.
    • boolean_expression is checked every time AFTER statement is run.
    • The example while can be written as the following for loop:
      for (i=3; i>0; i=i-1)
      {
        print(i)
      }
    • These loops do exactly the same thing.
  • Loops that don't end correctly (e.g. while(true)) repeat forever. This is called an infinite loop.
  • Function definitions are also statements. These will be discussed later.
