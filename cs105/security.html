Security

We explore common ways of securing computers from known attacks.

Firewalls

  • Attacks over a network, many backdoors, port scanning, etc. often work using specific ports.
  • Firewalls block all traffic going to or from certain ports.
    • For example, even if a computer was running a web server, if a firewall was set up to block port 80, then no computer will be able to connect to that web server.
    • The idea is to block ports which are known to be related to vulnerabilities and backdoors.
    • They can also be configured to block IP addresses, protocols, etc.
  • Firewalls can operate at the network level, or the computer level (often both).
    • Network level firewalls block a wide array of traffic coming in to a certain network.
    • Most IT departments have a network level firewall to block common attacks.
    • Each computer often has their own firewall to block more specific things.
    • Windows firewall, for example, blocks many incoming connections (it doesn't make sense for client machines to be listening for connections).
  • Many firewall rules can be bypassed.
    • For example, if a hacker owns a machine, and wants to open a backdoor, then instead of opening it at a blocked port, it can just be opened at an unblocked port.
    • Most firewalls do not block commonly-used ports such as those for DNS and HTTP.
    • To bypass IP blocks, one can simply use another IP, or a proxy server.
    • To bypass protocol blocks, a hacker can wrap one protocol within another (called tunneling).
  • Deep packet inspection allows for blocking specific threats, rather than simply blocking ports.
    • This is more difficult, because it requires some kind of intelligence in determining what kind of data a packet contains.
    • Requires more complicated rules.
    • Example: blocking access to certain websites (e.g. the great firewall).
    • Can be defeated by encryption, tunneling.

User Permissions

  • Every program runs as a "user".
    • Users actually represent a set of permissions, such as the ability to create a file, to read certain files, to create another user.
    • So programs can run as users which do not actually correspond to real people (in fact this is common).
  • Users restrict power.
    • For example, most people cannot install programs - only the administrator user can do that.
    • For programs, this is OK, as long as they have enough power to do what they need to do (e.g. web server is able to listen to and handle HTTP requests, etc.).
    • When programs get hacked, hackers will often try to use programs to do something that program is not supposed to do. Hence restrictions are needed to contain damage.
  • The principle of least privilege means giving each program the most restrictive user possible, while still allowing it to do its job.
    • For example, a web server does not need to create new users for any reason.
    • The principle dictates that we should not give the web server this ability.
    • Now if someone hacks the web server, and tries to use it to create a username and password (e.g. for a backdoor), the damage is contained.

Worms / Viruses / Antiviruses

  • Worms propagate from computer to computer.
    • Usually they exploit one or several known bugs, and are really specific.
    • Usually they constantly look for new targets, and try to infect them.
  • Viruses spread within a computer.
    • The purpose of a virus is to maintain access.
    • First, the virus tries to gain access by infecting a program.
    • If successful, then it tries to infect other programs. That way, just getting rid of the infected program won't get rid of the infection.
    • It may even attach itself to system processes, which make it very difficult to remove.
    • It may write itself to disk to maintain persistence.
  • Viruses are programs, so they often have predictable behavior.
    • Usually, they infect other programs in exactly the same way.
    • Antiviruses detect viral infections by looking for viral signatures, which describe how known viruses affect programs.
    • Antiviruses do this by scanning all program memory. This is why your computer is slow during a virus scan.
    • However, advanced viruses mutate themselves, so that they infect programs in different ways each time, making it very difficult to pinpoint a signature.

Encryption

  • Encryption can be used to hide from eavesdroppers.
  • Encrypted messages look garbled, and can only be deciphered by someone with the right encryption key.
  • Every encrypted connection requires a unique encryption key - this key is used to specify how the message will be encrypted.
  • There could an encryption key sent unencrypted at the beginning - an attacker could theoretically grab this, and decipher all future messages.
  • Techniques like Diffie-Hellman key exchange fix this.
